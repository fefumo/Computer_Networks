# 4 занятие, 27.03.2026

В рамках данного задания фактически необходимо повторить часть первой лабораторной работы по конфигурированию сети из трех устройств.

### План выполнения работы

#### Часть 1. Получение доступов

1. (опционально) Сгенерируйте ключи доступа по SSH с помощью ssh-keygen.
2. Отправьте публичный SSH ключ мне в личные сообщения на занятии.
3. Ожидайте получения доступов к серверам.
4. Подключитесь по индивидуальной инструкции к серверам и убедитесь в корректности их работы.

#### Часть 2. Конфигурирование сети из трех компьютеров, связанных bridge (2 балла)

1. Ознакомьтесь с информацией о конфигурации устройств в сети, постройте топологию этой сети.
2. Проверьте сетевую доступность всех устройств между собой по внутренним IP адресам Docker (каждый с каждым). Почему она именно такая?
3. Назначьте IP адреса на eth0 интерфейсах всех узлов, назначив две новые сети между тремя узлами: 192.168.1.0/24 и 192.168.2.0/24. Запрещено добавлять новые интерфейсы. IP адреса, выданные Docker игнорировать. Подумайте, что будет если удалить IP адрес с этого интерфейса?
4. Ознакомьтесь и настройте механизм IP Forwarding на центральном узле в топологии. При необходимости настройте таблицы маршрутизации на узлах, объясните необходимость / отсутствие необходимости настройки.
5. Проверьте сетевую доступность всех узлов между собой по IP, убедитесь, что все узлы доступны всем.
6. Зафиксируйте при помощи утилит захвата трафика (tcpdump / tshark / ...) последовательность запросов ответов по сети.
6. Составьте отчёт по пункту, в нем отразите последовательность действий для выполнения пункта, причины использования IP Forwarding для данной топологии, подтверждения из дампов трафика, ответы на вопросы и обоснования принятых решений. Всё важное, что было при выполнении работы.

Сеть между узлами (Docker-контейнерами) сконфигурирована следующим образом: 

```yaml
networks:
  group1_net:
    driver: bridge

services:
  server_a:
    # ...
    networks:
      group1_net:

  server_b:
    # ...
    networks:
      group1_net:

  server_c:
    # ...
    networks:
      group1_net:
```


#### Часть 3. Конфигурирование сети из трех компьютеров, связанных двумя bridge (2 балла)

1. Ознакомьтесь с информацией о конфигурации устройств в сети, постройте топологию этой сети.
2. Проверьте сетевую доступность всех устройств между собой по внутренним IP адресам Docker (каждый с каждым). Почему она именно такая?
3. Не добавляя новых интерфейсов на узлах, настройте две сети: 192.168.1.0/24 и 192.168.2.0/24. Рекомендую не удалять IP адреса с интерфейсов, которые выданы Docker.
4. Настройте таблицу маршрутизации на всех узлах.
5. Проверьте сетевую доступность всех узлов между собой по IP, убедитесь, что все узлы доступны всем.
6. Зафиксируйте при помощи утилит захвата трафика (tcpdump / tshark / ...) последовательность запросов ответов по сети.
6. Составьте отчёт по пункту, в нем отразите последовательность действий для выполнения пункта, подтверждения из дампов трафика, ответы на вопросы и обоснования принятых решений. Всё важное, что было при выполнении работы.

Сеть между узлами (Docker-контейнерами) сконфигурирована следующим образом: 

```yaml
networks:
  group2_net_1:
    driver: bridge
  group2_net_2:
    driver: bridge

services:
  server_a:
    networks:
      group2_net_1:

  server_b:
    networks:
      group2_net_1:
      group2_net_2:

  server_c:
    networks:
      group2_net_2:
```
